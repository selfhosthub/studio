CSS Hierarchy and Cascade Flow

CSS hierarchy determines how styles are applied to HTML elements. Understanding the specificity, inheritance, and cascading nature of CSS is essential for effective styling of web applications.

CSS Specificity

Specificity determines which CSS rule applies when multiple rules target the same element. Higher specificity rules override lower ones:

  • !important - Highest specificity, overrides all other styles
  • Inline Styles - Applied directly to elements with the style attribute
  • ID Selectors - Elements selected with #id
  • Class Selectors - Elements selected with .class names
  • Element Selectors - Elements selected by their tag name (p, div, h1)
  • Browser Defaults - The browser's default styling

CSS Variables and Inheritance

CSS variables (custom properties) are particularly useful for theming:

  • Define variables at :root level to make them globally available
  • Variables defined on parent elements are inherited by children
  • Override variables at any level for specific components
  • Use var(--name) to reference variables anywhere in your CSS

Cascading Nature

The "C" in CSS stands for "Cascading," which refers to the algorithm that determines which styles are applied:

  • Styles from different sources (browser defaults, user stylesheets, author stylesheets) are combined
  • When conflicting styles exist, specificity rules determine which one wins
  • If specificity is the same, the last declared style wins (source order matters)

Pro Tip: With CSS variables, you can define a theme at the root level and have those values cascade through your entire application, allowing for easy theme switching and consistent styling.
